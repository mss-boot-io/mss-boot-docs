## 与 HotGo 对比分析

### 能力对比

| 能力维度 | mss-boot-admin | HotGo | 差距分析 |
|----------|----------------|-------|----------|
| 系统配置 | 数据库存储多格式 | 配置中心 + 系统参数 | HotGo 配置中心更完善 |
| 通知系统 | Notice 表 + WebSocket | WebSocket 实时推送 | 能力相当 |
| 定时任务 | Default/K8s/Func 三种 Provider | CronJob + 插件任务 | mss-boot 更灵活 |
| 系统监控 | CPU/Memory/Disk/Network/Runtime | 服务监控 + 日志系统 | HotGo 有日志系统 |
| 统计查询 | Statistics 表 + 接口实现 | 统计报表 + 可视化 | HotGo 有可视化图表 |

### 主要差距

1. **日志系统**
   - HotGo 有完善的服务日志查询与管理界面
   - mss-boot-admin 缺少运行时日志查看能力

2. **可视化图表**
   - HotGo 有统计数据可视化图表支持
   - mss-boot-admin 统计数据仅 API 返回，无前端图表

3. **告警能力**
   - HotGo 支持监控告警配置
   - mss-boot-admin 缺少告警机制

## 详细任务规划

### 1. 运行时日志查看

**背景**

当前系统缺少运行时日志查看能力，运维排障需要登录服务器查看日志文件。

**方案**

1. 实现日志收集接口
2. 支持日志级别过滤
3. 支持关键词搜索
4. 支持时间范围筛选
5. 前端日志查看界面

**影响范围**

- 后端：`apis/log.go`（新增）、`models/log.go`（新增）
- 前端：`src/pages/Log/index.tsx`（新增）

**验收标准**

- [ ] 可查看最近 N 条日志
- [ ] 可按级别过滤（info/warn/error）
- [ ] 可按关键词搜索
- [ ] 可按时间范围筛选

### 2. 监控数据可视化

**背景**

当前监控数据仅 API 返回 JSON，无前端图表展示。

**方案**

1. 前端引入图表库（如 ECharts 或 Ant Design Charts）
2. 实现监控仪表盘页面
3. 支持 CPU/内存/磁盘/网络趋势图
4. 支持实时刷新

**影响范围**

- 前端：`src/pages/Dashboard/Monitor.tsx`（新增或修改）
- 依赖：图表库引入

**验收标准**

- [ ] CPU 使用率趋势图
- [ ] 内存使用趋势图
- [ ] 磁盘使用情况图表
- [ ] 网络流量图表
- [ ] 自动刷新间隔可配置

### 3. 告警规则配置

**背景**

当前系统无告警能力，问题依赖人工发现。

**方案**

1. 定义告警规则模型
2. 实现告警规则配置接口
3. 实现告警触发逻辑
4. 支持多渠道通知（系统通知/邮件/钉钉等）

**影响范围**

- 后端：`apis/alert.go`（新增）、`models/alert.go`（新增）
- 前端：`src/pages/Alert/index.tsx`（新增）

**验收标准**

- [ ] 可配置告警规则（指标/阈值/触发条件）
- [ ] 可配置通知渠道
- [ ] 告警触发后发送通知
- [ ] 告警历史可查询

## 实施优先级建议

### 高优先级（建议三期完成）

1. **运行时日志查看**
   - 运维基础能力
   - 对排障效率影响大

2. **监控数据可视化**
   - 用户体验提升明显
   - 前端改动为主，风险较低

### 中优先级（建议四期）

3. **告警规则配置**
   - 需要较多设计工作
   - 可先实现简单版本

4. **配置变更历史**
   - 对审计有帮助
   - 依赖配置模块扩展

### 低优先级（后续迭代）

5. **通知模板管理**
6. **任务执行分析**
7. **统计数据导出**
8. **智能运营能力**
