# 功能概览

mss-boot-admin 提供完整的后台管理系统功能，覆盖权限治理、组织管理、系统配置、运营支持等多个维度。

## 功能分层

系统功能分为五个核心层次：

1. **认证与访问控制** - 用户身份与权限管理
2. **组织与基础数据** - 组织结构与基础资料
3. **系统配置与运营** - 系统设置与通知任务
4. **国际化与多语言** - 多语言内容管理
5. **监控与可观测性** - 系统运行状态监控

---

# 1. 认证与访问控制

解决"谁可以进入系统、以什么身份工作、拥有哪些权限"的问题。

## 用户认证

### 登录方式

- **账号密码登录** - 标准登录方式
- **OAuth2 第三方登录** - GitHub、飞书等平台登录
- **Personal Access Token** - API 访问令牌

### 登录流程

1. 用户输入账号密码
2. 系统验证身份
3. 生成 JWT Token
4. 返回用户信息与权限数据

### Token 管理

- Token 有效期配置（默认 2 小时）
- Refresh Token 机制（自动续期）
- Token 撤销与黑名单管理

## 权限控制

### RBAC 权限模型

基于 Casbin 实现 RBAC（基于角色的访问控制）：

- 用户 - 实际操作者
- 角色 - 权限集合
- 权限 - 具体操作许可

权限结构：

```
用户 -> 角色 -> 菜单权限 + API权限
```

### 菜单权限

控制用户可见菜单：

- 菜单可见性
- 操作按钮权限
- 页面元素权限

### API 权限

控制接口访问：

- API 路径权限
- HTTP 方法权限
- 动态权限策略

### 数据权限

控制数据访问范围：

- 全部数据
- 本部门数据
- 自定义范围

## 个人中心

### 用户资料

- 个人信息维护
- 头像上传
- 联系方式管理

### 密码管理

- 修改密码
- 安全验证
- 密码强度检查

### Token 管理

- 生成 Personal Access Token
- 查看 Token 列表
- 撤销 Token

# 2. 组织与基础数据

解决"系统中有哪些人、部门、岗位，以及业务中通用的选项数据如何维护"的问题。

## 用户管理

### 用户信息

| 字段 | 说明 |
|------|------|
| 用户名 | 登录账号，唯一标识 |
| 昵称 | 显示名称 |
| 邮箱 | 联系邮箱 |
| 手机 | 联系电话 |
| 部门 | 所属组织 |
| 岗位 | 职位信息 |
| 角色 | 权限角色 |
| 状态 | 启用/禁用 |

### 用户操作

- 新增用户

- 分配角色
- 启用/禁用用户

## 部门管理

- 多层级部门树
- 部门负责人
- 部门联系信息

### 部门信息

| 字段 | 说明 |

| 状态 | 启用/禁用 |

维护组织中的岗位信息：

- 岗位名称与编码
- 岗位层级
- 岗位说明

| 字段 | 说明 |
|------|------|
| 岗位编码 | 唯一标识 |
| 岗位名称 | 显示名称 |
| 岗位层级 | 职级高低 |
| 状态 | 启用/禁用 |

## 角色管理

### 角色信息

维护系统角色：

| 字段 | 说明 |
|------|------|
| 角色名称 | 显示名称 |

- API 权限分配

| 类型 | 说明 |
|------|------|

| 图标 | 菜单图标 |
| 显示顺序 | 排序权重 |

# 5. 监控与可观测性

提供系统运行状态监控能力。

## 系统监控

### 服务器监控

监控服务器状态：

- CPU 使用率
- 内存使用率
- 磁盘使用率
- 网络流量

### 运行信息

- 系统版本
- 运行时间
- Goroutine 数量
- 数据库连接数

### 监控图表

- CPU 趋势图
- 内存趋势图
- 实时监控数据

## 可观测性

### Prometheus 指标

暴露监控指标：

- HTTP 请求指标
- 数据库指标
- 系统指标

指标路径：`/metrics`

### 健康检查

健康检查路径：

- `/health` - 健康状态
- `/ready` - 就绪状态

### pprof 性能分析

性能分析端点：

- CPU 分析
- 内存分析
- Goroutine 分析
- 阻塞分析

# 功能使用建议

## 权限配置流程

1. 创建菜单结构
2. 创建角色
3. 为角色分配菜单权限和 API 权限
4. 创建用户
5. 为用户分配角色

## 组织配置流程

1. 创建部门结构
2. 创建岗位信息
3. 创建用户并关联部门和岗位

## 运营配置流程

1. 配置系统参数
2. 发布通知公告
3. 配置定时任务
4. 配置告警规则
