# 角色提示词索引（mss-boot-docs）

## 目录说明

- 本目录存放 `mss-boot-docs` 相关角色提示词与跨角色协作文档。
- 当前聚焦开发测试角色和五角色协作总览。
- 文件命名遵循小写 kebab-case，中文文档使用 `.zh-CN.md` 后缀。

## 当前文件

1. `dev-test-role.zh-CN.md`
   - 开发测试角色定义。
   - 负责联调验证、测试用例、回归结果和文档沉淀。

2. `role-collaboration-map.zh-CN.md`
   - 五角色协作总览。
   - 描述角色边界、协作流转、交接触发点和推荐使用方式。

## 使用建议

- OpenCode 默认应由用户先与 leader 沟通，再由 leader 决定何时引入开发测试角色和协作总览。
- 当任务需要联调、验收、回归、文档沉淀或交接说明时，leader 应优先引入开发测试角色。
- 当任务需要明确五个角色的协作关系、交接顺序和沟通机制时，应优先阅读协作总览。
